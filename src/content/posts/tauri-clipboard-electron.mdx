sections:
  - type: h2
    content: 왜 Tauri를 선택했는가
  - type: p
    content: 클립보드 관리 앱을 만들 때 Electron이 가장 먼저 떠올랐습니다. 하지만 Electron 앱의 기본 바이너리 크기는 50MB를 훌쩍 넘깁니다. 클립보드 유틸리티처럼 항상 백그라운드에서 실행되는 앱에게 이건 치명적입니다.
  - type: h2
    content: Tauri + Rust의 핵심 구조
  - type: p
    content: Tauri는 프론트엔드로 시스템 WebView를 사용하고, 백엔드는 Rust로 작성합니다. 클립보드 훅킹 같은 OS 수준 작업은 Rust의 arboard 크레이트가 담당합니다.
  - type: code
    content: |
      // Rust 백엔드: 클립보드 이벤트 감지
      #[tauri::command]
      fn watch_clipboard(app: AppHandle) {
          let mut ctx = arboard::Clipboard::new().unwrap();
          loop {
              if let Ok(text) = ctx.get_text() {
                  app.emit("clipboard-change", text).unwrap();
              }
              std::thread::sleep(Duration::from_millis(300));
          }
      }
